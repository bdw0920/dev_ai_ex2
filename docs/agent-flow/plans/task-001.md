# TASK-001: SNMP 장비 목록 조회 API 최소 기능 추가

## 목표
SNMP 기반 장비의 목록을 조회할 수 있는 최소한의 REST API를 제공한다.

## 범위 (in_scope)
- GET /api/v1/devices/snmp 엔드포인트 구현
- JSON 응답 형식: "[{\"id\": \"1\", \"name\": \"Router-01\", \"ip\": \"192.168.1.1\", \"type\": \"router\"}]"
- 쿼리 파라미터 없음
- 장비 데이터는 하드코딩된 예시를 사용
- 요청에 응답 성공 상태 코드 200 반환
- 간단한 개발 문서 작성

## 제외 범위 (out_of_scope)
- SNMP 수집 로직 구현
- 장비 데이터 저장소 (DB, 파일 등) 연동
- 인증/인가 메커니즘 포함
- 정렬, 필터링, 페이징 기능
- 프론트엔드 UI
- 테스트 커버리지
- 실시간 구독 기능

## 완료 조건
- `plans/task-001.md` 파일이 생성되어야 한다
- `handoff_to` 필드가 `coder`로 지정되어야 한다
- 범위가 명확하고 과도하게 확장되지 않아야 한다
- out_of_scope 항목이 명시되어야 한다

## 테스트 요구사항
- 단일 API 엔드포인트가 존재하는지 확인
- HTTP GET 요청에 대해 200 코드와 예상 리스폰스가 반환되는지 확인

## 참고사항
- 기존 코드베이스에 영향을 주지 않도록 주의한다
- 임시 데이터를 사용한다
- 미리 정의된 API 스펙에 맞추어야 한다
